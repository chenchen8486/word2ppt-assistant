# Phase 5: 数据完整性验证与修复机制

## 任务 1: JSON数据序列完整性验证 (`validate_repair.py`)
- **需求**: 验证 `_extracted.json` 文件中的题目序列是否完整，检测是否存在题目编号缺失的情况。
- **逻辑**:
  - 提取所有题目的数字编号（支持普通数字如"1", "2"和带括号格式如"22（1）", "22（2）"）
  - 检查从最小编号到最大编号之间是否存在间隙
  - 特别关注之前经常缺失的题目编号（如8, 9, 10, 11, 12, 13, 16, 18, 19, 20, 21, 23等）

## 任务 2: 自动修复缺失题目 (`verify_and_fix_questions.py`)
- **需求**: 自动识别并修复缺失的题目条目，确保数据连续性。
- **逻辑**:
  - 读取现有 `_extracted.json` 文件，识别已存在的题目编号
  - 从备份源（如`test_chunks.json`）获取缺失题目的内容
  - 以正确的JSON结构添加缺失的题目（包括`type`, `number`, `content`, `answer`, `analysis`字段）
  - 按题目编号重新排序整个数组

## 任务 3: 数据顺序重组 (`reorder_extracted.py`)
- **需求**: 修正数据顺序，确保每个context后面跟着与之相关的questions，而不是将所有contexts和questions分离。
- **逻辑**:
  - 根据`test_chunks.json`的原始顺序，重构`test_extracted.json`中的项目顺序
  - 按照题目逻辑分组，每个context（一、二、三等）后跟随相关的问题
  - 确保数据流与原始文档结构保持一致

## 任务 4: 错误数据处理与恢复 (`recreate_extracted.py`)
- **需求**: 从包含错误信息的损坏JSON文件中恢复有效数据
- **逻辑**:
  - 检测并识别包含`"error"`字段的损坏条目
  - 从错误信息字符串中解析可能的JSON数据
  - 重建完整且结构正确的数据集合
  - 确保数据的完整性和有效性

## 任务 5: 综合数据校验 (`comprehensive_check.py`)
- **需求**: 全面校验数据文件的完整性、结构正确性、错误处理和数据顺序
- **逻辑**:
  - 检查是否存在错误条目（包含`error`字段）
  - 验证所有项目的基本结构（`type`, `number`, `content`等字段）
  - 统计各类项目数量（contexts vs questions）
  - 验证编号完整性（检查是否包含所有预期的contexts和question编号）
  - 验证question编号的连续性（1-24，包括带括号的子题如22(1), 22(2)）
  - 检查数据顺序（每个context后跟随相关的questions）
  - 验证字段完整性（question项目包含answer和analysis）

## 任务 6: 数据质量保证流程
- **验证机制**: 在PPT生成前自动验证数据完整性
- **修复机制**: 自动或手动触发修复脚本以恢复丢失的数据
- **排序机制**: 确保数据顺序符合原始文档的逻辑结构
- **错误处理机制**: 识别和处理包含错误信息的条目
- **监控机制**: 提供清晰的日志输出显示哪些题目被修复

## 验证标准
- 验证脚本能正确检测题目序列中的缺失
- 修复脚本能正确添加缺失题目并保持数据结构一致性
- 重排序脚本能正确按照context-question的逻辑顺序排列数据
- 错误处理机制能正确识别和修复损坏的数据
- 综合校验脚本能全面验证数据的完整性、结构和顺序
- 修复后PPT生成流程正常工作且包含完整的题目序列
